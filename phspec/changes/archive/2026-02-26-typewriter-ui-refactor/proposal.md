# 提案：Love Diary 机械打字机风格 UI 重构

## 动机

当前 Love Diary 页面为通用现代风格（渐变、圆角卡片、照片墙），辨识度与情感张力不足。希望通过**机械打字机**隐喻重构整页视觉与交互：整页即一台打字机，点击某个主题（时间线回忆或照片）时，在「纸面」区域用**打印动画**逐字/逐行展示内容，强化「写日记、一页一页翻」的沉浸感。页面设计将借助 ui-ux-pro-max 等 UI/UX 最佳实践，确保复古风格下仍保持清晰层级、可访问性与一致性。

## 变更内容

- 整体布局重构为「打字机」隐喻：顶盖/品牌区、主题入口区（按键或色带式）、中央纸面展示区、底座。
- 主题入口：时间线条目与照片墙项均可点击，视觉上呈现为打字机按键或纸卷/色带标签。
- 纸面内容展示：点击主题后，在唯一的主内容区（纸面）内展示；文字采用逐字/逐行打字动画与光标，图片采用「从出纸口推出」或「贴纸」式出现。
- 视觉风格：暖灰/米色背景、米白纸面与细纸纹、深灰/墨色字体、衬线体、单色强调（深棕/墨绿等）、少量打字机细节装饰（色带轴、卷纸边等）。
- 可选：打字敲击声效（可关闭）、打印结束的节奏感；动效可配置速度或跳过（无障碍）。
- 保留现有能力：统计（天数、回忆数、照片数）、添加回忆（FAB + 模态表单）、时间线与照片数据来源不变；仅改变呈现方式与交互隐喻。

## 能力

### 新增能力

- `love-diary-typewriter-ui`：Love Diary 机械打字机风格界面。涵盖整页布局隐喻、主题入口（时间线/照片）的视觉与点击行为、纸面区域的打印动画（文字逐字/逐行、图片出纸/贴图）、复古视觉规范（色彩、字体、纸纹、装饰），以及可选音效与无障碍控制。设计实施时遵循 ui-ux-pro-max 等 UI/UX 最佳实践。

### 修改的能力

- （无。当前仓库 `phspec/specs/` 下无既有规范；本次为单页应用 UI 重构，不涉及既有能力的需求级变更。）

## 影响

- **主要文件**：Next 应用下的页面与布局（如 app/page.tsx、app/layout.tsx）及本变更将新增或修改的组件与样式（结构、样式、脚本/交互逻辑）。
- **样式**：由当前现代扁平风改为复古打字机风（CSS 变量、布局、字体、背景与纸面纹理、动画 keyframes）。
- **交互与脚本**：主题点击统一驱动纸面内容更新；新增打字/出纸动画逻辑、可选音效与无障碍控制（如跳过动画、减少动效）。
- **依赖**：字体（如衬线体、等宽数字）、可选音效资源；无新增运行时依赖。
- **设计参考**：页面设计需借助 ui-ux-pro-max（或等效 UI/UX 规范），确保层次、对比度、焦点与可访问性达标。
