# 设计：Love Diary 机械打字机 UI

## 背景

- **现状**：当前 Love Diary 为 Next 应用（如 app/page.tsx 等），含 Hero、统计卡片、时间线、照片墙、FAB + 添加回忆模态；样式为现代扁平风（Archivo/Space Grotesk、青绿系、圆角卡片），照片点击仅 console 输出。
- **约束**：在现有 Next 应用内实现，可不依赖后端；设计遵循 ui-ux-pro-max 等 UI/UX 规范（层级、对比度、焦点、可访问性）。
- **干系人**：最终用户（阅读/浏览回忆）、实施者（前端）。

## 目标 / 非目标

**目标：**

- 整页呈现为一台机械打字机（顶盖、主题入口区、纸面区、底座），视觉与交互隐喻一致。
- 主题入口（时间线条目、照片项）可点击，点击后在中央纸面区用「打印」动画展示内容（文字逐字/逐行 + 光标，图片出纸/贴图）。
- 复古视觉系统落地：暖灰/米色背景、米白纸面与纸纹、衬线体与墨色、单色强调、少量装饰。
- 可选打字声效与无障碍控制（跳过/减速动效），不破坏现有添加回忆与统计能力。

**非目标：**

- 不引入框架或构建步骤；不改变数据来源（仍为内联/现有 DOM 结构）。
- 不做服务端、PWA 或离线持久化；不实现多用户或实时协作。

## 决策

### 1. 布局与 DOM 结构

- **决策**：用单页分区（顶盖区、按键/入口区、纸面区、底座区）的语义区块 + CSS 实现打字机外形；纸面区为单一内容容器，点击主题时只更新该容器内容并触发动画，不新开页面或路由。
- **理由**：单页不新开路由，保持简单；单一纸面避免多「纸」状态同步问题。替代方案（多 tab/多纸面）会增加状态与视觉噪音，不符合「一台打字机一张纸」的隐喻。

### 2. 文字打印动画实现方式

- **决策**：用 JavaScript 按字符或按行推进，将内容写入纸面 DOM，并同步更新一个「光标」节点（如 `|` 或竖线）的位置；每字/每行间隔用 `setTimeout` 或 `requestAnimationFrame` + 可配置 delay 控制。
- **理由**：逐字/逐行便于做「打字头」节奏与可跳过逻辑；纯 CSS 无法按内容长度驱动逐字出现。替代方案（CSS 遮罩 + 动画宽度）难以处理换行与多段，且可访问性（屏幕阅读器一次拿到全文）不如渐进暴露 + 可选「立即显示」按钮。

### 3. 图片展示方式

- **决策**：先打 1～2 行说明文字（如标题/日期），再在纸面内以「从上方缝口推出」的动画展示图片（用 CSS `clip-path` 或 `transform` 做从上往下 reveal），图片外框加轻微纸纹/泛黄边框以贴合纸面。
- **理由**：图片不适合逐像素「打印」；出纸隐喻与打字机一致。替代方案（整图淡入）太普通；「贴纸」式（图+边框）可作为简化版或低性能设备的降级。

### 4. 主题入口与数据绑定

- **决策**：时间线条目与照片项保留现有 DOM 结构，增加 `data-*` 属性（如 `data-type="timeline"`、`data-index` 或内联的标题/日期/正文/图片 URL），点击时读取并传入统一的「打印」控制器，由控制器负责清空纸面、选择打字或出图流程并执行动画。
- **理由**：不引入状态库；数据与展示仍来自当前页面/组件（或 data-\* / props），便于与现有添加回忆逻辑（动态插入时间线节点）兼容。替代方案（独立 JSON 数据源）会重复维护两处结构。

### 5. 视觉与主题系统

- **决策**：用 CSS 变量集中定义复古色板（背景、纸面、墨色、强调色、边框）、字体（衬线正文、等宽数字/日期）、纸面纹理（细粒度背景图或 SVG pattern），并保留一处「装饰」类（色带轴、卷纸边等）便于后续微调。
- **理由**：变量便于做主题切换或暗色变体；纹理用 CSS 可避免额外图片请求。字体选用 Lora / Libre Baskerville / Noto Serif SC 等衬线体以贴合打字机/印刷感。

### 6. 音效与无障碍

- **决策**：打字敲击声为可选，通过一个全局开关（如设置或首次提示）开启/关闭；提供「跳过动画」或「立即显示」控制，并在 `prefers-reduced-motion: reduce` 时自动缩短或跳过逐字动画。
- **理由**：音效增强沉浸但可能干扰；尊重系统减少动效偏好是无障碍基本要求。替代方案（无跳过）会损害可访问性与部分用户体验。

## 风险 / 权衡

| 风险                       | 缓解措施                                                               |
| -------------------------- | ---------------------------------------------------------------------- |
| 长文逐字动画耗时过长       | 提供「立即显示」按钮；可配置每字 delay 上限或改为逐行模式              |
| 纸面区在窄屏上空间不足     | 纸面区使用响应式内边距与字号，必要时单列、按键区可折叠或收起到「抽屉」 |
| 装饰与纹理增加视觉噪音     | 装饰控制在少数元素、低对比；纸纹使用极淡透明度                         |
| 与现有添加回忆表单风格割裂 | 模态沿用复古色板与衬线体，与纸面区统一为同一视觉系统                   |

## 迁移计划

- **部署**：更新 Next 应用（如 `app/` 下页面与组件），无分阶段发布需求。
- **回滚**：保留原文件副本或通过版本控制回退即可；无数据迁移。
